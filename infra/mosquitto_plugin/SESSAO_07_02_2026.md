## Requisito Solicitado

Implementar controle de acesso no plugin Mosquitto com **duas camadas de validação**:

### Camada 1: Validação de ACL (Access Control List)
- Fazer split no client_id do subscriber para extrair userId
- Verificar se o userId tem permissão para acessar o tópico
- Baseado em lista de wildcards MQTT (`+` e `#`)

### Camada 2: Validação de Consent
- Extrair serviceId do tópico (formato: `aop/<serviceId>/...`)
- Verificar se o serviceId está na lista `accessConsent` do usuário
- Conformidade com GDPR

## Análise do Sistema Existente

### Access Consent no GingaDistrib

**Arquivo:** `user-files/userData.json`

```json
{
  "users": [
    {
      "id": "c3167a18-5dc5",
      "name": "Anonymous",
      "accessConsent": ["fe2481ea-5d44-4225-884b-504782636c3a", "0"]
    }
  ]
}
```

**Objetivo do accessConsent:**
- Controlar quais serviços (broadcasters/canais) têm permissão para acessar dados do perfil do usuário
- Conformidade com GDPR (General Data Protection Regulation)
- Usuário autoriza explicitamente cada serviço

**Fluxo no GingaDistrib:**
1. Usuário seleciona um perfil
2. Serviço (broadcaster) solicita acesso aos dados do perfil
3. Sistema verifica se `serviceId` está no array `accessConsent` do usuário
4. Se estiver → permite acesso aos dados/arquivos do usuário
5. Se não estiver → nega acesso (erro 305)

**Implementação atual (TypeScript):**
```typescript
// ccws/src/modules/user-api/expression.ts
function getConsentExpression(service: string, path: string): string {
    let exp = `users['${service}' in accessConsent and avatar='${path}'] != null`;
    return exp;
}

// ccws/src/modules/user-api/service.ts
async function checkConsent(fpath: string): Promise<boolean> {
    let expression = jsonata(exp.getConsentExpression(data.currentService, fpath));
    let result = await expression.evaluate(data.users);
    return result;
}
```

## Notas Técnicas

### Wildcards MQTT
- `+` - Match de um único nível
  - `sensor/+/temperature` match `sensor/room1/temperature`
  - Não match `sensor/room1/data/temperature`
  
- `#` - Match de múltiplos níveis (deve ser último)
  - `aop/#` match `aop/1/apps` e `aop/1/2/path`
  - `aop/+/#` match `aop/1/apps` mas não `aop/users`

### Formato Client ID
- **Obrigatório:** `user_<userId>`
- Exemplo: `user_c3167a18-5dc5`
- O plugin extrai o userId removendo o prefixo `user_`

### Reason Codes MQTT v5
- `0` - Success
- `135` - Not authorized (usado pelo plugin)
- `153` - Payload format invalid (não usado - desconecta cliente)

### Performance
- Validação em memória (JSON carregado na inicialização)
- Complexidade: O(n) para ACL, O(m) para consent
- Sem impacto significativo na latência
